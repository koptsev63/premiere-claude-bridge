>

> > «Возьми все .MTS из папки Grave Stakes/Videos, импортируй в проект, логни через motion и audio, дай мне HTML с миниатюрами»
>

>
> Не для финального монтажа. Для черновой сборки и логирования — да. Это работа ассистента-монтажёра, которая раньше съедала день.
>

> Готов отдать в бету 20 первым желающим (Mac + Premiere 2024+). Дальше — открытый код на GitHub + платные skill-pack'и для специфических жанров.

## Вариант C — максимально нетривиальный, с провокацией

> Adobe в описании своего AI-коннектора пишет «Premiere included».
>
> Я открыл — там **нет ни одного инструмента** для управления десктопным Premiere Pro. Четыре облачные AI-фичи, которые работают с загруженным видео в браузере. К программе на твоём Маке они не имеют отношения. Они сами в документации пишут «для обрезки по таймкоду — используйте Adobe Premiere Pro».
>
> Ну, окей. Я взял и сделал.
>
> Open-source мост Claude ↔ Premiere Pro. Промт `«импортируй вот эту папку с 108 клипами, склей секвенцию по такому-то cutlist'у, поставь маркеры на ключевые эмоции, выгрузи в AME»` — выполняется за минуту.
>
> Под капотом: MCP-сервер на Node + CEP-расширение для Premiere. Доступны все операции ExtendScript — импорт, бины, секвенции, in/out клипов, маркеры, экспорт. Плюс операционка для монтажа на основе «В мгновение ока» Мерча: правило шести (эмоция > история > ритм > eye-trace > 2D > 3D), теория моргания, decisive moment.
>
> Что я честно НЕ делаю:
> — fine cut (это работа человека, не AI)
> — выбор «правильного» дубля по микровыражению (мой потолок — стоп-кадры, не движение)
> — драматургию из ничего (надо задать структуру явно)
>
> Что делаю хорошо:
> — логирование за минуты вместо часов
> — структурный assembly из ваших selects
> — технические задачи (gaps, нормализация, маркеры, AME)
> — несколько вариантов одной сборки за секунды
>
> Релиз через 2 недели. До этого — закрытый бета-тест на 20 человек. Кто хочет — пишите. Mac + Premiere Pro 2024+.
